dbca  [-silent | -progressOnly | -customCreate] {<command> <options> }  | { [<command> [options] ] -responseFile  <response file > } [-continueOnNonFatalErrors <true | false>]
Pour plus de détails, reportez-vous à la documentation.
Vous pouvez saisir l'une des commandes suivantes :

Créez une base de données en indiquant les paramètres suivants :
	-createDatabase
		-templateName <nom d'un modèle existant dans l'emplacement par défaut ou chemin complet du modèle>
		[-cloneTemplate]
		-gdbName <nom global de base de données>
		[-RACOneNode 
			-RACOneNodeServiceName  <Nom du service à créer pour la base de données RAC One Node.>]
		[-policyManaged | -adminManaged <Base de données gérée par une stratégie ou par un administrateur. Par défaut, la base de données est gérée par un administrateur>]
			[-createServerPool <Pour créer un pool de serveurs qui sera utilisé par la base de données à créer>]
			[-force <Pour forcer la création d'un pool de serveurs lorsque les serveurs libres adéquats ne sont pas disponibles. Cela peut affecter la base de données déjà en cours d'exécution>]
			-serverPoolName <Un nom de pool de serveurs en cas de création et une liste de noms de pool de serveurs séparés par des virgules en cas d'utilisation>
			-[cardinality <Indiquez la cardinalité du pool de serveurs à créer. La valeur par défaut est le nombre de noeuds qualifiés>]
		[-sid <préfixe de l'identificateur système (SID) de base de données>]
		[-sysPassword <mot de passe de l'utilisateur SYS>]
		[-systemPassword <mot de passe de l'utilisateur SYSTEM>]
		[-emConfiguration <CENTRAL|LOCAL|ALL|NONE>
			-dbsnmpPassword <mot de passe de l'utilisateur DBSNMP>
			-sysmanPassword <mot de passe de l'utilisateur SYSMAN>
			[-hostUserName <Nom utilisateur d'hôte pour le travail de sauvegarde EM>
			 -hostUserPassword <Mot de passe d'utilisateur d'hôte pour le travail de sauvegarde EM>
			 -backupSchedule <Programmation de la sauvegarde quotidienne au format hh:mm>]
			[-centralAgent <Répertoire d'origine Oracle Home de l'agent central Enterprise Manager>]]
		[-disableSecurityConfiguration <ALL|AUDIT|PASSWORD_PROFILE|NONE>
		[-datafileDestination <répertoire de destination pour tous les fichiers de base de données> |  -datafileNames <fichier texte contenant des objets de base de données, tels que les fichiers de contrôle, les tablespaces, les fichiers de journalisation et le fichier SPFILE, avec mappings des noms de fichier correspondants du raw device sous la forme nom=valeur.>]
		[-redoLogFileSize <taille de chaque fichier de journalisation en méga-octets>]
		[-recoveryAreaDestination <répertoire de destination pour tous les fichiers de récupération>]
		[-datafileJarLocation  <emplacement du fichier de données jar, utilisé uniquement pour la création de la base de données clone>]
		[-storageType < CFS | ASM > 
			[-asmsnmpPassword     <Mot de passe ASMSNMP pour la surveillance ASM>]
			 -diskGroupName   <nom du groupe de disques de la zone de base de données>
			 -recoveryGroupName       <nom du groupe de disques de la zone de récupération>
		[-nodelist <noms de noeud séparés par une virgule (pour la base de données)>]
		[-characterSet <jeu de caractères de la base de données>]
		[-nationalCharacterSet  <jeu de caractères national de la base de données>]
		[-registerWithDirService <true | false> 
			-dirServiceUserName    <nom utilisateur pour le service d'annuaire>
			-dirServicePassword    <mot de passe pour le service d'annuaire >
			-walletPassword    <mot de passe pour le portefeuille de base de données >]
		[-listeners  <liste des processus d'écoute avec lesquels doit être configurée la base de données>]
		[-variablesFile   <nom de fichier pour la paire variable-valeur des variables utilisées dans le modèle>]]
		[-variables  <liste des paires nom=valeur séparées par une virgule>]
		[-initParams <liste des paires nom=valeur séparées par une virgule>]
		[-sampleSchema  <true | false> ]
		[-memoryPercentage <pourcentage de la mémoire physique d'Oracle>]
		[-automaticMemoryManagement ]
		[-totalMemory <mémoire allouée pour Oracle en Mo>]
		[-databaseType <MULTIPURPOSE|DATA_WAREHOUSING|OLTP>]]

Configurez une base de données en indiquant les paramètres suivants :
	-configureDatabase
		-sourceDB    <Nom unique de la base de données RAC et SID pour la base de données mono-instance>
		[-sysDBAUserName     <nom utilisateur avec des privilèges SYSDBA>
		 -sysDBAPassword     <mot de passe pour le nom utilisateur sysDBAUserName>]
		[-registerWithDirService|-unregisterWithDirService|-regenerateDBPassword <true | false> 
			-dirServiceUserName    <nom utilisateur pour le service d'annuaire>
			-dirServicePassword    <mot de passe pour le service d'annuaire >
			-walletPassword    <mot de passe pour le portefeuille de base de données >]
		[-disableSecurityConfiguration <ALL|AUDIT|PASSWORD_PROFILE|NONE>
		[-enableSecurityConfiguration <true|false>
		[-emConfiguration <CENTRAL|LOCAL|ALL|NONE>
			-dbsnmpPassword <mot de passe de l'utilisateur DBSNMP>
			-sysmanPassword <mot de passe de l'utilisateur SYSMAN>
			[-hostUserName <Nom utilisateur d'hôte pour le travail de sauvegarde EM>
			 -hostUserPassword <Mot de passe d'utilisateur d'hôte pour le travail de sauvegarde EM>
			 -backupSchedule <Programmation de la sauvegarde quotidienne au format hh:mm>]
			[-centralAgent <Répertoire d'origine Oracle Home de l'agent central Enterprise Manager>]]


Créez un modèle à partir d'une base de données existante en indiquant les paramètres suivants :
	-createTemplateFromDB
		-sourceDB    <format du service : <hôte>:<port>:<sid>>
		-templateName      <nouveau nom de modèle>
		-sysDBAUserName     <nom utilisateur avec des privilèges SYSDBA>
		-sysDBAPassword     <mot de passe pour le nom utilisateur sysDBAUserName>
		[-maintainFileLocations <true | false>]


Créez un modèle clone à partir d'une base de données existante en indiquant les paramètres suivants :
	-createCloneTemplate
		-sourceSID    <SID de la base de données source>
		-templateName      <nouveau nom de modèle>
		[-sysDBAUserName     <nom utilisateur avec des privilèges SYSDBA>
		 -sysDBAPassword     <mot de passe pour le nom utilisateur sysDBAUserName>]
		[-maintainFileLocations <true | false>]
		[-datafileJarLocation       <répertoire dans lequel doivent être placés les fichiers de données au format compressé>]

Générez des scripts de création de base de données en indiquant les paramètres suivants :
	-generateScripts
		-templateName <nom d'un modèle existant dans l'emplacement par défaut ou chemin complet du modèle>
		-gdbName <nom global de base de données>
		[-scriptDest       <destination pour tous les fichiers de script>]

Supprimez une base de données en indiquant les paramètres suivants :
	-deleteDatabase
		-sourceDB    <Nom unique de la base de données RAC et SID pour la base de données mono-instance>
		[-sid    <nom d'instance (instance_name) locale de la base de données source>]
		[-sysDBAUserName     <nom utilisateur avec des privilèges SYSDBA>
		 -sysDBAPassword     <mot de passe pour le nom utilisateur sysDBAUserName>]

Ajoutez une instance à une base de données de cluster en indiquant les paramètres suivants :
	-addInstance 
		-gdbName <nom global de base de données>
		-nodelist <nom de noeud de la nouvelle instance à ajouter>
		[-instanceName <nom de la nouvelle instance à ajouter>]
		[-sysDBAUserName     <nom utilisateur avec des privilèges SYSDBA>]
		 -sysDBAPassword     <mot de passe pour le nom utilisateur sysDBAUserName>
		[-updateDirService <true | false> 
			-dirServiceUserName    <nom utilisateur pour le service d'annuaire>
			-dirServicePassword    <mot de passe pour le service d'annuaire >]

Supprimez une instance d'une base de données de cluster en indiquant les paramètres suivants :
	-deleteInstance 
		-gdbName <nom global de base de données>
		-instanceName <nom de l'instance à enlever>
		[-nodelist <nom de noeud de l'instance à enlever>]
		[-sysDBAUserName     <nom utilisateur avec des privilèges SYSDBA>]
		 -sysDBAPassword     <mot de passe pour le nom utilisateur sysDBAUserName>
		[-updateDirService <true | false> 
			-dirServiceUserName    <nom utilisateur pour le service d'annuaire>
			-dirServicePassword    <mot de passe pour le service d'annuaire >]
Demandez de l'aide en indiquant les options suivantes : -h | -help
